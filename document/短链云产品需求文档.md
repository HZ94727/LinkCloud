## 1. 项目背景

### 1.1 业务背景

短链接服务在营销推广、电商带货、私域运营等场景中已成为基础工具。

- **营销人员**：需要在短信、海报、社群里投放链接，同时追踪效果
- **微商/电商卖家**：需要短链接防屏蔽、美观、可统计
- **知识付费**：需要保护付费内容，防止链接被随意传播

目前市场上存在以下情况：
- 免费工具功能受限（生成条数少、无统计）
- 国内大厂短链接服务（如新浪短链）已关停
- 现有方案分散，没有统一好用的产品

### 1.2 当前问题

| 问题 | 描述 |
|------|------|
| 链接太长 | 在短信、微信中不美观，占用字符多 |
| 无法统计 | 发了链接不知道多少人点、什么时候点 |
| 容易屏蔽 | 长链接容易被微信/QQ拦截 |
| 无法保护 | 资料链接分享后无法控制谁看 |
| 工具缺失 | 市面上缺少好用、稳定、便宜的短链接工具 |

### 1.3 机会点

| 机会 | 说明 |
|------|------|
| 市场缺口 | 大厂短链服务关停，留下市场空白 |
| 付费意愿 | 营销人员愿意为数据付费 |
| 技术成熟 | 高性能后端方案成熟，可支撑高并发 |
| 商业模式清晰 | 免费+付费订阅模式已被验证可行 |

### 1.4 项目目标

**核心目标：** 打造一款好用、稳定、可商业化的短链接服务平台

**具体目标：**
- 提供稳定可靠的短链接跳转服务
- 提供清晰的数据统计和分析功能
- 通过免费+付费模式实现商业变现
- 积累用户，建立品牌认知

### 1.5 项目价值

| 价值类型 | 说明 |
|----------|------|
| 用户价值 | 解决链接太长、无法统计、易被屏蔽的痛点 |
| 商业价值 | 通过会员订阅产生持续收入 |
| 品牌价值 | 建立技术品牌，可延伸至其他营销工具 |

## 2. 范围说明

### 2.1 本期范围

本期主要建设短链接服务的**核心业务闭环**，支持用户从注册登录到生成短链接、基础统计的完整使用流程。

**本期范围包括：**

| 模块 | 功能       | 说明                     |
| :--- | :--------- | :----------------------- |
| 用户 | 注册       | 邮箱注册                 |
| 用户 | 登录       | 用户名/邮箱 + 密码登录   |
| 用户 | 退出登录   | 安全退出                 |
| 短链 | 生成短链接 | 长链接转短链接           |
| 短链 | 点击计数   | 记录每个短链接的点击次数 |
| 管理 | 链接列表   | 查看所有生成的短链接     |
| 管理 | 删除链接   | 删除不需要的链接         |
| 统计 | 点击量展示 | 展示每个链接的总点击量   |

### 2.2 本期不包含

| 第三方登录（微信/QQ） | 简化 V1.0，后期增加     |
| 批量生成              | 简化 V1.0，后期增加     |
| 自定义域名            | 简化 V1.0，后期增加     |
| 数据导出              | 简化 V1.0，后期增加     |

## 3. 目标用户

### 3.1 用户画像

**典型用户一：营销人员小王**

- 年龄：28岁                                           
- 职业：某教育公司市场部专员                           
- 工作内容：负责短信营销、社群运营                         
- 痛点：发出去的链接不知道多少人点，无法向领导汇报效果 
- 需求：需要知道点击量、点击时间、来源渠道             
- 付费意愿：愿意为数据报表付费，方便做复盘                 

**用户二：微商小李**
- 年龄：26岁                                           
- 职业：全职微商，卖护肤品化妆品等                       
- 工作内容：每天发朋友圈、维护微信群                         
- 痛点：长链接经常被微信屏蔽，影响生意
- 需求：需要短链接、防屏蔽、美观            
- 付费意愿：愿意为稳定服务付费，链接失效就是损失  

**用户四：知识付费卖家小陈**
- 年龄：26岁                                           
- 职业：在线课程卖家                       
- 工作内容：售卖编程课程、资料包                      
- 痛点：课程资料链接被转发，未付费用户也能看
- 需求：需要密码保护、有效期控制            
- 付费意愿：愿意为内容保护功能付费 

### 3.2 用户场景

| 场景 | 具体描述 |
|------|----------|
| 短信营销 | 将短链接放入短信内容，追踪短信点击率 |
| 朋友圈推广 | 生成短链接发朋友圈，防止被微信屏蔽 |
| 社群分享 | 在微信群发送短链接，统计群成员点击情况 |
| 内容保护 | 为资料链接设置密码，仅付费用户可访问 |
| 限时活动 | 设置链接有效期，活动结束后自动失效 |

## 5. 功能需求

### 5.1 用户注册

#### 5.1.1 功能说明

- 用户通过邮箱注册账号，成为平台的正式用户。注册后可以登录系统，使用短链接生成、链接管理、数据统计等功能。

#### 5.1.2 使用角色

| 角色       | 说明               |
| :--------- | :----------------- |
| 未注册访客 | 首次使用平台的用户 |

#### 5.1.3 触发方式

| 触发方式                   | 说明                       |
| :------------------------- | :------------------------- |
| 点击登录页的“立即注册”链接 | 从登录页跳转到注册页       |
| 直接访问注册页面           | 通过浏览器直接访问注册页面 |

#### 5.1.4 前置条件

| 条件       | 说明                         |
| :--------- | :--------------------------- |
| 网络正常   | 能够访问平台服务             |
| 邮箱未注册 | 当前使用的邮箱尚未注册过账号 |

#### 5.1.5  输入项

| 字段     | 是否必填 | 说明                       | 规则                                          |
| :------- | :------- | :------------------------- | :-------------------------------------------- |
| 邮箱     | 是       | 用户注册邮箱，作为登录账号 | 符合邮箱格式；长度不超过100字符；不可重复注册 |
| 密码     | 是       | 登录密码                   | 长度6-20位；至少包含字母和数字；区分大小写    |
| 确认密码 | 是       | 再次输入密码               | 必须与密码字段一致                            |
| 验证码   | 是       | 邮箱验证码                 | 6位数字；5分钟内有效                          |

#### 5.1.6 处理逻辑

**阶段一：获取验证码**

| 步骤 | 动作       | 说明                               |
| :--- | :--------- | :--------------------------------- |
| 1    | 接收参数   | 获取 email                         |
| 2    | 参数校验   | 校验 email 合法性                  |
| 3    | 频率校验   | 检查该邮箱60秒内是否已发送过验证码 |
| 4    | 生成验证码 | 生成6位随机数字                    |
| 5    | 存储缓存   | 存入缓存并设置到期时间             |
| 6    | 发送邮件   | 调用邮件服务发送验证码到邮箱       |
| 7    | 返回响应   | 返回发送成功                       |

**阶段二：提交注册**

| 步骤 | 动作       | 说明                                                      |
| :--- | :--------- | :-------------------------------------------------------- |
| 1    | 接收参数   | 获取 email、password、captcha                             |
| 2    | 参数校验   | 校验 email 格式、password 长度(6-20位)、captcha 为6位数字 |
| 3    | 验证码校验 | 从缓存中查询验证码，校验是否一致且未过期                  |
| 4    | 唯一性校验 | 查询数据库，确认 email 未被注册                           |
| 5    | 密码加密   | 使用加密算法对 password 加密                              |
| 6    | 创建用户   | 插入用户相关信息到数据库中                                |
| 7    | 生成Token  | 生成 JWT Token 并设置一定时间的有效期                     |
| 8    | 返回响应   | 返回成功及 Token                                          |

#### 5.1.7 输出结果

| 结果类型       | 说明                                         |
| :------------- | :------------------------------------------- |
| 获取验证码成功 | 弹出成功提示，停留在当前页面                 |
| 获取验证码失败 | 弹出失败提示，停留在当前页面                 |
| 注册成功       | 弹出成功提示，自动跳转到登录页               |
| 注册失败       | 弹出错误提示，停留在当前页面，保留已填写内容 |

#### 5.1.8 页面与交互要求

| 交互       | 要求                                           |
| :--------- | :--------------------------------------------- |
| 实时校验   | 输入邮箱时实时校验格式；输入密码时实时校验强度 |
| 实时提示   | 格式错误时，输入框下方红字提示                 |
| 按钮状态   | 必填项未填或格式错误时，注册按钮置灰           |
| 获取验证码 | 点击后按钮进入60秒倒计时，倒计时内不可重复点击 |
| 提交加载态 | 点击注册后，按钮显示“注册中”，禁止重复提交     |
| 成功处理   | 收到成功响应后，跳转到控制台首页               |
| 失败处理   | 根据错误码显示对应提示，按钮恢复可点击状态     |
| 回车提交   | 在任意输入框按回车键可触发生成                 |

#### 5.1.9 业务规则

| 规则         | 说明                       |
| :----------- | :------------------------- |
| 邮箱唯一性   | 一个邮箱只能注册一个账号   |
| 密码加密     | 密码不得明文存储           |
| 验证码有效期 | 5分钟有效，过期需重新获取  |
| 验证码频率   | 同一邮箱60秒内只能获取一次 |

#### 5.1.10 异常场景

| 场景           | 处理方式 | 返回文案                       |
| :------------- | :------- | :----------------------------- |
| 邮箱格式错误   | 返回错误 | “邮箱格式不正确”               |
| 密码不符合规则 | 返回错误 | “密码需6-20位，包含字母和数字” |
| 验证码错误     | 返回错误 | “验证码错误”                   |
| 验证码过期     | 返回错误 | “验证码已过期，请重新获取”     |
| 发送频率过高   | 返回错误 | “操作太频繁，请60秒后再试”     |
| 邮箱已注册     | 返回错误 | “该邮箱已注册”                 |
| 系统错误       | 返回错误 | “系统繁忙，请稍后重试”         |

#### 5.1.11 验收标准

| 编号 | 验收项         | 预期结果                                 |
| :--- | :------------- | :--------------------------------------- |
| 1    | 获取验证码     | 输入正确邮箱，点击后收到验证码邮件       |
| 2    | 验证码频率     | 60秒内重复点击，按钮禁用并倒计时         |
| 3    | 正常注册       | 填写正确信息后，成功注册并跳转到首页     |
| 4    | 邮箱格式校验   | 输入非法邮箱时，前端提示错误，无法提交   |
| 5    | 邮箱重复校验   | 使用已注册邮箱时，后端返回“邮箱已注册”   |
| 6    | 密码规则校验   | 密码不符合规则时，前端提示错误           |
| 7    | 密码一致性校验 | 两次密码不一致时，前端提示错误           |
| 8    | 验证码校验     | 输入错误验证码时，后端返回“验证码错误”   |
| 9    | 验证码过期     | 使用过期验证码时，后端返回“验证码已过期” |
| 10   | 注册成功入库   | 数据库中新增对应用户记录                 |
| 11   | 密码加密存储   | 数据库中密码字段为加密值                 |

### 5.2 用户登录

#### 5.2.1 功能说明

- 已注册用户通过邮箱和密码登录系统，获取使用短链接服务的权限。登录后用户可以访问个人短链接列表、生成新短链接、查看统计数据等。

#### 5.2.1 功能说明使用角色

| 角色       | 说明                 |
| :--------- | :------------------- |
| 已注册用户 | 已完成邮箱注册的用户 |

#### 5.2.3 触发方式

| 触发方式             | 说明                     |
| :------------------- | :----------------------- |
| 访问登录页           | 直接访问登录页面 URL     |
| 点击“登录”链接       | 从注册页或首页跳转       |
| 未登录访问需鉴权页面 | 系统自动跳转到登录页     |
| 退出登录后           | 用户主动退出后返回登录页 |

#### 5.2.4 前置条件

| 条件       | 说明               |
| :--------- | :----------------- |
| 网络正常   | 能够访问平台服务   |
| 已注册账号 | 用户已完成邮箱注册 |

#### 5.2.5 输入项

| 字段     | 是否必填 | 说明               | 规则               |
| :------- | :------- | :----------------- | :----------------- |
| 邮箱     | 是       | 注册时使用的邮箱   | 符合邮箱格式       |
| 密码     | 是       | 注册时设置的密码   | 长度6-20位         |
| 记住密码 | 否       | 勾选后保持登录状态 | 布尔值，默认未勾选 |

#### 5.2.6 处理逻辑

| 步骤 | 动作         | 说明                                                   |
| :--- | :----------- | :----------------------------------------------------- |
| 1    | 接收参数     | 获取 email、password、rember                           |

| 4    | 用户存在校验 | 查询数据库判断用户是否存在                             |
| 5    | 密码校验     | 对比输入的密码与存储的加密密码，判断是否一致           |
| 6    | 账号状态校验 | 检查账号是否被禁用                                     |
| 7    | 生成 Token   | 生成 JWT Token， 设置一定的有效期，remember=true/false |
| 8    | 记录登录日志 | 记录登录时间、IP（异步）                               |
| 9    | 返回响应     | 返回成功及 Token                                       |

#### 5.2.7 输出结果

| 结果类型 | 说明                                 |
| :------- | :----------------------------------- |
| 登录成功 | 跳转到控制台首页，显示用户短链接列表 |
| 登录失败 | 显示错误提示，停留在登录页           |

#### 5.2.8 页面与交互要求

| 交互     | 要求                                       |
| :------- | :----------------------------------------- |
| 实时校验 | 输入邮箱时实时校验格式                     |
| 按钮状态 | 必填项未填时，登录按钮置灰                 |
| 加载态   | 点击登录后，按钮显示“登录中”，禁止重复提交 |
| 成功处理 | 跳转到控制台首页                           |
| 失败处理 | 显示错误提示，按钮恢复可点击状态           |
| 回车登录 | 在密码框按回车键可触发登录                 |
| 密码显示 | 密码框右侧提供“眼睛”图标，可切换明文/密文  |
| 记住密码 | 勾选后，下次访问自动填充账号信息           |

#### 5.2.9 业务规则

| 规则         | 说明                                             |

| 单点登录     | 同一账号同一时间只能在一处登录，新登录踢掉旧登录 |
| 异常IP提醒   | 检测到异地登录时，发送邮件提醒用户               |
| 密码加密     | 密码不得明文传输，前端加密后传输                 |
| Token有效期  | 记住密码7天，否则24小时                          |

#### 5.2.10 异常场景

| 密码错误     | 返回错误 | “邮箱或密码错误”                   |

| 系统错误     | 返回错误 | “系统繁忙，请稍后重试”             |

#### 5.2.11 验收标准

### 5.4 短链接跳转

#### 5.4.1 功能说明

当用户（访客）点击短链接时，系统根据短码查询对应的长链接，记录访问日志，然后通过302重定向将访客跳转到原始长链接地址。

#### 5.4.2 使用角色

| 角色 | 说明                           |
| :--- | :----------------------------- |
| 访客 | 任何点击短链接的人，不需要登录 |

#### 5.4.3 触发方式

| 触发方式       | 说明                         |
| :------------- | :--------------------------- |
| 点击短链接     | 在任何地方点击短链接         |
| 浏览器直接访问 | 在浏览器地址栏输入短链接地址 |

#### 5.4.4 前置条件

| 条件         | 说明                         |
| :----------- | :--------------------------- |
| 短链接存在   | 短码在系统中有对应的映射记录 |
| 短链接未禁用 | 链接状态为“启用”             |
| 短链接未过期 | 未超过设置的有效期（如有）   |
| 密码正确     | 如有密码保护，需输入正确密码 |

#### 5.4.5 输入项

| 字段 | 是否必填 | 说明                 | 规则               |
| :--- | :------- | :------------------- | :----------------- |
| 短码 | 是       | URL路径中的短码      | 6-8位字母数字组合  |
| 密码 | 否       | 密码保护链接需要输入 | 用户输入的访问密码 |

#### 5.4.6 处理逻辑

| 步骤 | 动作         | 说明                                       |
| :--- | :----------- | :----------------------------------------- |
| 1    | 接收请求     | 从 URL 路径中提取短码                      |
| 2    | 参数校验     | 校验短码非空                               |
| 3    | 查询缓存     | 从缓存中查询短码对应的链接信息             |
| 4    | 缓存命中判断 | 缓存命中则直接使用，未命中则查询数据库     |
| 5    | 查询数据库   | 从数据库查询短码对应的链接信息             |
| 6    | 短码存在校验 | 短码不存在时返回404                        |
| 7    | 状态校验     | 检查短码是否为启用状态                     |
| 8    | 有效期校验   | 检查短码是否已过期，如果短码有过期时间     |
| 9    | 密码校验     | 检查请求携带的密码是否正确                 |
| 10   | 写入缓存     | 将查询结果写入缓存，设置到期时间           |
| 11   | 记录访问日志 | 记录请求方信息，包括IP、设备类型、访问时间 |
| 12   | 返回重定向   | 返回重定向的响应                           |

#### 5.4.7 输出结果

| 场景            | 返回内容              |
| :-------------- | :-------------------- |
| 成功            | 302，重定向到长链接   |
| 需要密码        | 200，返回密码输入页面 |
| 短码不存在      | 404，返回错误页面     |
| 链接已禁用/过期 | 403，返回错误页面     |

#### 5.4.8 页面与交互要求

| 交互       | 要求                                     |
| :--------- | :--------------------------------------- |
| 密码页面   | 简洁的密码输入表单，居中显示，移动端适配 |
| 密码输入框 | 自动聚焦，支持回车提交                   |
| 密码错误   | 显示错误提示，不清空密码框               |
| 404页面    | 显示“链接不存在或已被删除”               |
| 过期页面   | 显示“该链接已超过有效期”                 |

#### 5.4.9 业务规则

| 规则         | 说明                               |
| :----------- | :--------------------------------- |
| 重定向类型   | 使用302临时重定向                  |
| 缓存策略     | 热点链接缓存，设置到期时间         |
| 日志异步     | 访问日志异步写入，不影响跳转性能   |
| 防刷机制     | 同一IP每秒超过10次请求时，临时限流 |
| 密码错误限制 | 同一短码连续5次密码错误，锁定5分钟 |

#### 5.4.10 异常场景

| 场景         | 处理方式              | 展示内容                          |
| :----------- | :-------------------- | :-------------------------------- |
| 短码不存在   | 返回404页面           | “链接不存在或已被删除”            |
| 短链接已禁用 | 返回禁用页面          | “该链接已被禁用”                  |

#### 5.4.11 验收标准

| 编号 | 验收项     | 预期结果                             |
| :--- | :--------- | :----------------------------------- |
| 1    | 正常跳转   | 点击短链接后，1秒内跳转到原始链接    |
| 2    | 短码不存在 | 访问不存在的短码，显示404页面        |

## 8. 数据需求

### 8.1 用户数据

用于标识系统内的账号主体，是登录、生成短链接、管理链接等业务的基础对象。

#### 关键字段

| 字段名称     | 说明                 | 来源       | 用途                           | 是否持久化 |
| :----------- | :------------------- | :--------- | :----------------------------- | :--------- |
| 用户ID       | 用户唯一标识         | 服务端生成 | 用户身份识别、关联其他业务数据 | 是         |
| 邮箱         | 用户登录账号         | 用户输入   | 登录、注册、接收验证码         | 是         |
| 昵称         | 用户显示名称         | 用户输入   | 页面展示                       | 是         |
| 密码哈希     | 加密后的登录密码     | 服务端加密 | 登录验证                       | 是         |
| 账号状态     | 正常/封禁            | 服务端维护 | 判断是否允许登录和使用         | 是         |
| 注册时间     | 账号创建时间         | 服务端生成 | 记录用户创建时间               | 是         |
| 最近登录时间 | 最近一次成功登录时间 | 系统记录   | 审计、状态查看                 | 是         |

#### 数据规则

- 邮箱在系统内必须唯一；
- 账号状态影响登录和使用权限；
- 用户数据为系统核心主数据，必须长期保存。

### 8.2 短链接数据

系统的核心业务数据，记录长链接与短码的映射关系。

#### 关键字段

| 字段名称 | 说明                    | 来源       | 用途               | 是否持久化 |
| :------- | :---------------------- | :--------- | :----------------- | :--------- |
| 短码     | 短链接的唯一标识（6位） | 服务端生成 | 跳转时查询映射关系 | 是         |
| 长链接   | 用户输入的原始网址      | 用户输入   | 跳转目标地址       | 是         |

## 9. 错误码/提示规则

### 9.1 设计目标

为保证短链接服务平台在注册、登录、生成短链接、跳转、链接管理等场景下的异常处理方式统一，系统需建立统一的错误码与提示规则，用于：

1. 统一业务异常表达；
2. 降低用户理解成本；
3. 提升开发、测试、客服对异常问题的定位效率；
4. 避免不同页面、不同模块提示不一致。

### 9.2 错误码分类规则

系统错误码按业务模块进行分类，主要包括：

| 分类           | 说明                             |
| :------------- | :------------------------------- |
| 通用错误码     | 适用于公共失败场景               |
| 注册错误码     | 适用于注册、验证码等场景         |
| 登录错误码     | 适用于登录、会话失效等场景       |
| 短链接错误码   | 适用于生成短链接、跳转等场景     |
| 链接管理错误码 | 适用于编辑、删除、状态变更等场景 |
| 统计错误码     | 适用于数据查询场景               |
| 安全错误码     | 适用于限流、权限校验等场景       |

### 9.3 提示规则

1. 用户可见提示应优先使用业务语言，不直接暴露技术细节；
2. 同类错误在不同页面中应保持提示文案一致；
3. 输入校验类错误应优先在前端拦截并提示；
4. 服务端异常、网络异常、超时等公共异常应使用统一提示文案；
5. 安全相关错误应避免暴露具体校验逻辑和内部实现；
6. 错误提示应能够帮助用户理解问题并完成下一步操作。

### 9.4 提示方式定义

| 场景类型             | 提示方式                      | 说明                           |
| :------------------- | :---------------------------- | :----------------------------- |
| 输入项为空或格式错误 | 输入框下方提示 / 页面局部提示 | 用于引导用户修正输入           |
| 操作失败             | 页面错误提示                  | 展示业务操作失败原因           |
| 权限校验失败         | 弹窗提示 / 页面高优先级提示   | 用于提示无权限、需登录等       |
| 网络异常 / 服务异常  | 页面统一错误提示              | 不暴露内部异常信息             |
| 成功操作             | Toast / 状态提示              | 如生成成功、修改成功、删除成功 |

1. 不直接向用户展示数据库异常、程序异常堆栈或内部错误描述；

3. 对于服务异常、网络异常、超时等公共异常，统一使用标准文案，避免页面间提示不一致；
4. 对于配额用完、链接禁用、链接过期等状态，应提示明确的业务结果及建议动作（如升级套餐）；
5. 输入校验类错误（如邮箱格式、密码规则）应在前端实时提示，减少服务端请求；
6. 成功操作（生成、修改、删除）应使用 Toast 轻提示，不阻塞用户操作。
