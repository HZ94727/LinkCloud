## 1. 项目背景

### 1.1 业务背景

短链接服务在营销推广、电商带货、私域运营等场景中已成为基础工具。

- **营销人员**：需要在短信、海报、社群里投放链接，同时追踪效果
- **微商/电商卖家**：需要短链接防屏蔽、美观、可统计
- **知识付费**：需要保护付费内容，防止链接被随意传播

目前市场上存在以下情况：
- 免费工具功能受限（生成条数少、无统计）
- 国外工具（如 Bitly）访问慢、不符合国内使用习惯
- 国内大厂短链接服务（如新浪短链）已关停
- 现有方案分散，没有统一好用的产品

### 1.2 当前问题

| 问题 | 描述 |
|------|------|
| 链接太长 | 在短信、微信中不美观，占用字符多 |
| 无法统计 | 发了链接不知道多少人点、什么时候点 |
| 容易屏蔽 | 长链接容易被微信/QQ拦截 |
| 无法保护 | 资料链接分享后无法控制谁看 |
| 工具缺失 | 市面上缺少好用、稳定、便宜的短链接工具 |

### 1.3 机会点

| 机会 | 说明 |
|------|------|
| 市场缺口 | 大厂短链服务关停，留下市场空白 |
| 付费意愿 | 营销人员愿意为数据付费 |
| 技术成熟 | 高性能后端方案成熟，可支撑高并发 |
| 商业模式清晰 | 免费+付费订阅模式已被验证可行 |

### 1.4 项目目标

**核心目标：** 打造一款好用、稳定、可商业化的短链接服务平台

**具体目标：**
- 提供稳定可靠的短链接跳转服务
- 提供清晰的数据统计和分析功能
- 通过免费+付费模式实现商业变现
- 积累用户，建立品牌认知

### 1.5 项目价值

| 价值类型 | 说明 |
|----------|------|
| 用户价值 | 解决链接太长、无法统计、易被屏蔽的痛点 |
| 商业价值 | 通过会员订阅产生持续收入 |
| 品牌价值 | 建立技术品牌，可延伸至其他营销工具 |

## 2. 范围说明

### 2.1 本期范围（V1.0）

**本期范围包括：**

| 模块 | 功能       | 说明                     |
| :--- | :--------- | :----------------------- |
| 用户 | 注册       | 邮箱注册                 |
| 用户 | 登录       | 用户名/邮箱 + 密码登录   |
| 用户 | 退出登录   | 安全退出                 |
| 短链 | 生成短链接 | 长链接转短链接           |

| 统计 | 点击量展示 | 展示每个链接的总点击量   |

## 3. 目标用户

### 3.1 用户画像

**典型用户一：营销人员小王**

- 年龄：28岁                                           
- 职业：某教育公司市场部专员                           
- 工作内容：负责短信营销、社群运营                         
- 痛点：发出去的链接不知道多少人点，无法向领导汇报效果 
- 需求：需要知道点击量、点击时间、来源渠道             
- 付费意愿：愿意为数据报表付费，方便做复盘                 

**用户二：微商小李**
- 年龄：26岁                                           
- 职业：全职微商，卖护肤品化妆品等                       
- 工作内容：每天发朋友圈、维护微信群                         
- 痛点：长链接经常被微信屏蔽，影响生意
- 需求：需要短链接、防屏蔽、美观            
- 付费意愿：愿意为稳定服务付费，链接失效就是损失  

**用户四：知识付费卖家小陈**
- 年龄：26岁                                           
- 职业：在线课程卖家                       
- 工作内容：售卖编程课程、资料包                      
- 痛点：课程资料链接被转发，未付费用户也能看
- 需求：需要密码保护、有效期控制            
- 付费意愿：愿意为内容保护功能付费 

### 3.2 用户场景

| 场景 | 具体描述 |
|------|----------|
| 短信营销 | 将短链接放入短信内容，追踪短信点击率 |
| 朋友圈推广 | 生成短链接发朋友圈，防止被微信屏蔽 |
| 社群分享 | 在微信群发送短链接，统计群成员点击情况 |
| 内容保护 | 为资料链接设置密码，仅付费用户可访问 |
| 限时活动 | 设置链接有效期，活动结束后自动失效 |

## 5. 功能需求

### 5.1 用户注册

#### 5.1.1 功能说明

- 用户通过邮箱注册账号，成为平台的正式用户。注册后可以登录系统，使用短链接生成、链接管理、数据统计等功能。

#### 5.1.2 使用角色

| 角色       | 说明               |
| :--------- | :----------------- |
| 未注册访客 | 首次使用平台的用户 |

#### 5.1.3 触发方式

| 触发方式                   | 说明                       |
| :------------------------- | :------------------------- |
| 点击登录页的“立即注册”链接 | 从登录页跳转到注册页       |
| 直接访问注册页面 URL       | 通过浏览器直接访问注册页面 |

#### 5.1.4 前置条件

| 条件       | 说明                                   |
| :--------- | :------------------------------------- |
| 网络正常   | 能够访问平台服务                       |
| 未被注册   | 当前使用的邮箱尚未注册过账号           |
| 浏览器支持 | 支持现代浏览器（Chrome、Edge、Safari） |

#### 5.1.5  输入项

| 字段     | 是否必填 | 说明                       | 规则                                          |
| :------- | :------- | :------------------------- | :-------------------------------------------- |
| 邮箱     | 是       | 用户注册邮箱，作为登录账号 | 符合邮箱格式；长度不超过100字符；不可重复注册 |
| 密码     | 是       | 登录密码                   | 长度6-20位；至少包含字母和数字；区分大小写    |
| 确认密码 | 是       | 再次输入密码               | 必须与密码字段一致                            |
| 验证码   | 是       | 邮箱验证码                 | 6位数字；5分钟内有效                          |

#### 5.1.6 处理逻辑

**正常流程：**

| 步骤 | 动作       | 说明                                                      |
| :--- | :--------- | :-------------------------------------------------------- |
| 1    | 接收参数   | 获取 email、password、captcha                             |
| 2    | 参数校验   | 校验 email 格式、password 长度(6-20位)、captcha 为6位数字 |
| 3    | 验证码校验 | 校验和缓存存储是否一致且未过期                            |
| 4    | 唯一性校验 | 确认 email 未被注册                                       |
| 5    | 密码加密   | 使用加密算法对 password 加密                              |
| 6    | 创建用户   | 将用户信息插入用户表                                      |
| 7    | 生成Token  | 生成 JWT Token（有效期24小时）                            |
| 8    | 清除验证码 | 删除缓存中的验证码，一次性使用                            |
| 9    | 返回响应   | 返回成功及 Token                                          |

#### 5.1.7 输出结果

| 结果类型 | 说明                                         |
| :------- | :------------------------------------------- |
| 注册成功 | 弹出成功提示，自动跳转到登录页               |
| 注册失败 | 弹出错误提示，停留在当前页面，保留已填写内容 |

#### 5.1.8 页面与交互要求

**页面布局：**

- 注册表单居中显示
- 包含：邮箱输入框、密码输入框、确认密码输入框、验证码输入框、获取验证码按钮
- 底部：用户协议勾选框、注册按钮、跳转登录链接

**交互要求：**

| 交互           | 要求                                                 |
| :------------- | :--------------------------------------------------- |
| 实时校验       | 邮箱格式、密码强度可实时校验提示                     |
| 获取验证码按钮 | 点击后进入60秒倒计时，倒计时内不可重复点击           |
| 注册按钮状态   | 必填项未填写时置灰不可点击                           |
| 加载态         | 点击注册后按钮显示“注册中，请稍后再试”，防止重复提交 |
| 成功提示       | 使用 Toast 或弹窗提示“注册成功，即将跳转登录页”      |
| 自动跳转       | 注册成功后3秒自动跳转到登录页                        |
| 失败提示       | 在对应字段下方或顶部显示错误信息                     |

**用户协议：**

- 默认未勾选
- 点击链接可查看完整《用户协议》和《隐私政策》
- 未勾选时点击注册按钮，提示“请阅读并同意用户协议”

#### 5.1.9 业务规则

| 规则           | 说明                              |
| :------------- | :-------------------------------- |
| 邮箱唯一性     | 一个邮箱只能注册一个账号          |
| 密码加密       | 密码不得明文存储，需使用加密算法  |
| 验证码有效期   | 验证码5分钟内有效，过期需重新获取 |
| 验证码频率限制 | 同一邮箱60秒内只能获取一次验证码  |

#### 5.1.10 异常场景

| 场景               | 处理方式                 | 提示文案                       |
| :----------------- | :----------------------- | :----------------------------- |
| 邮箱格式错误       | 输入框标红，显示错误提示 | “请输入正确的邮箱地址”         |
| 邮箱已注册         | 提示用户，可跳转登录     | “该邮箱已注册，可直接登录”     |
| 密码不符合规则     | 输入框标红，显示规则     | “密码需6-20位，包含字母和数字” |
| 两次密码不一致     | 确认密码框标红           | “两次输入的密码不一致”         |
| 验证码错误         | 提示重新输入             | “验证码错误，请重新输入”       |
| 验证码过期         | 提示重新获取             | “验证码已过期，请重新获取”     |
| 验证码发送频率过高 | 按钮禁用，显示倒计时     | “请60秒后再试”                 |
| 未勾选用户协议     | 注册按钮置灰或点击提示   | “请阅读并同意用户协议”         |
| 网络异常           | 提示网络问题             | “网络异常，请稍后重试”         |
| 系统错误           | 提示稍后重试             | “系统繁忙，请稍后重试”         |

#### 5.1.11 验收标准

| 编号 | 验收项             | 预期结果                                     |
| :--- | :----------------- | :------------------------------------------- |
| 1    | 正常注册           | 填写正确信息后，成功注册并跳转到登录页       |
| 2    | 邮箱格式校验       | 输入非法邮箱时，提示错误，无法提交           |
| 3    | 邮箱重复校验       | 使用已注册邮箱时，提示“邮箱已注册”           |
| 4    | 密码规则校验       | 密码不符合规则时，提示错误                   |
| 5    | 密码一致性校验     | 两次密码不一致时，提示错误                   |
| 6    | 验证码必填         | 不填验证码时，提示错误                       |
| 7    | 验证码正确性       | 输入错误验证码时，提示“验证码错误”           |
| 8    | 验证码过期         | 使用过期验证码时，提示“验证码已过期”         |
| 9    | 验证码频率限制     | 60秒内重复点击获取验证码时，按钮禁用并倒计时 |
| 10   | 用户协议校验       | 未勾选用户协议时，无法注册                   |
| 11   | 注册成功后数据入库 | 数据库中新增对应用户记录                     |
| 12   | 密码加密存储       | 数据库中密码字段为加密值，非明文             |

### 5.2 用户登录

#### 5.2.1 功能说明

- 已注册用户通过邮箱和密码登录系统，获取使用短链接服务的权限。登录后用户可以访问个人短链接列表、生成新短链接、查看统计数据等。

#### 5.2.1 功能说明使用角色

| 角色       | 说明                 |
| :--------- | :------------------- |
| 已注册用户 | 已完成邮箱注册的用户 |

#### 5.2.3 触发方式

| 触发方式             | 说明                     |
| :------------------- | :----------------------- |
| 访问登录页           | 直接访问登录页面 URL     |
| 点击“登录”链接       | 从注册页或首页跳转       |
| 未登录访问需鉴权页面 | 系统自动跳转到登录页     |
| 退出登录后           | 用户主动退出后返回登录页 |

#### 5.2.4 前置条件

| 条件       | 说明                                   |
| :--------- | :------------------------------------- |
| 网络正常   | 能够访问平台服务                       |
| 已注册账号 | 用户已完成邮箱注册                     |
| 浏览器支持 | 支持现代浏览器（Chrome、Edge、Safari） |

#### 5.2.5 输入项

| 字段     | 是否必填 | 说明               | 规则                            |
| :------- | :------- | :----------------- | :------------------------------ |
| 邮箱     | 是       | 注册时使用的邮箱   | 符合邮箱格式；长度不超过100字符 |
| 密码     | 是       | 注册时设置的密码   | 长度6-20位；包含字母和数字      |
| 记住密码 | 否       | 勾选后保持登录状态 | 布尔值，默认未勾选              |

#### 5.2.6 处理逻辑

**正常流程：**

| 步骤 | 动作                   | 说明                           |
| :--- | :--------------------- | :----------------------------- |
| 1    | 用户填写登录信息       | 填写邮箱、密码                 |
| 2    | 用户可选勾选“记住密码” | 勾选后延长登录有效期           |
| 3    | 用户点击“登录”按钮     | 提交登录请求                   |
| 4    | 系统校验输入           | 校验邮箱格式、密码非空         |
| 5    | 系统查询用户           | 根据邮箱查询用户表             |
| 6    | 系统验证密码           | 对比输入的密码与存储的加密密码 |
| 7    | 系统生成登录凭证       | 生成 Token 或 Session          |
| 8    | 系统记录登录日志       | 记录登录时间、IP               |
| 9    | 系统返回登录成功       | 跳转到用户控制台首页           |

**密码验证流程：**

| 步骤 | 动作               | 说明                         |
| :--- | :----------------- | :--------------------------- |
| 1    | 获取用户输入的密码 | 明文密码                     |
| 2    | 对输入密码进行加密 | 使用与注册时相同的加密算法   |
| 3    | 对比加密后的密码   | 与数据库中存储的密码哈希对比 |
| 4    | 返回验证结果       | 一致则通过，不一致则失败     |

**记住密码逻辑：**

| 勾选状态         | 登录有效期 | 说明                   |
| :--------------- | :--------- | :--------------------- |
| 勾选“记住密码”   | 7天        | 关闭浏览器后仍保持登录 |
| 未勾选“记住密码” | 24小时     | 关闭浏览器后需重新登录 |

#### 5.2.7 输出结果

| 结果类型 | 说明                                 |
| :------- | :----------------------------------- |
| 登录成功 | 跳转到控制台首页，显示用户短链接列表 |
| 登录失败 | 显示错误提示，停留在登录页           |

#### 5.2.8 页面与交互要求

**页面布局：**

### 5.3 生成短链接

#### 5.3.1 功能说明

- 用户输入一个长链接（原始网址），系统将其转换为一个简短的短链接。生成的短链接可以分享到短信、微信、邮件等渠道，点击后会自动跳转到原始长链接地址。

**解决的问题：**

- 长链接在短信、微信中占用字符多、不美观
- 长链接容易被微信/QQ等平台屏蔽
- 为后续数据统计提供基础（每个短链接独立追踪）

| 点击侧边栏的“新建短链接”菜单 | 导航菜单入口 |
| 控制台首页的快捷生成框       | 快捷入口     |

### 5.4 短链接跳转（核心功能）

#### 5.4.1 功能说明

当用户（访客）点击短链接时，系统根据短码查询对应的长链接，记录访问日志，然后通过302重定向将访客跳转到原始长链接地址。

**解决的问题：**

- 实现短链接到长链接的转换
- 记录每次访问的详细信息（IP、设备、来源、时间）
- 为数据统计提供原始数据

#### 5.4.2 使用角色

| 角色 | 说明                           |
| :--- | :----------------------------- |
| 访客 | 任何点击短链接的人，不需要登录 |

#### 5.4.3 触发方式

| 触发方式       | 说明                         |
| :------------- | :--------------------------- |
| 点击短链接     | 在任何地方点击短链接         |
| 浏览器直接访问 | 在浏览器地址栏输入短链接地址 |
| 扫码访问       | 扫描短链接对应的二维码       |

#### 5.4.4 前置条件

| 条件         | 说明                         |
| :----------- | :--------------------------- |
| 短链接存在   | 短码在系统中有对应的映射记录 |
| 短链接未禁用 | 链接状态为“启用”             |
| 短链接未过期 | 未超过设置的有效期（如有）   |
| 密码正确     | 如有密码保护，需输入正确密码 |

#### 5.4.5 输入项

| 字段 | 是否必填 | 说明                 | 规则               |
| :--- | :------- | :------------------- | :----------------- |
| 短码 | 是       | URL路径中的短码      | 6-8位字母数字组合  |
| 密码 | 否       | 密码保护链接需要输入 | 用户输入的访问密码 |

#### 5.4.6 处理逻辑

**正常流程：**

| 步骤 | 动作               | 说明                                         |
| :--- | :----------------- | :------------------------------------------- |
| 1    | 访客点击短链接     | 浏览器发起GET请求                            |
| 2    | 系统解析短码       | 从URL路径中提取短码                          |
| 3    | 系统查询短链接     | 根据短码查询映射关系（先查缓存，再查数据库） |
| 4    | 系统校验链接状态   | 检查是否存在、是否启用、是否过期             |
| 5    | 系统校验密码       | 如有密码，检查是否携带正确密码               |
| 6    | 系统记录访问日志   | 记录IP、设备类型、来源、时间（异步）         |
| 7    | 系统返回302重定向  | 返回302状态码和Location头                    |
| 8    | 浏览器跳转到原地址 | 访客看到原始网站页面                         |

**缓存策略：**

| 步骤 | 动作          | 说明                               |
| :--- | :------------ | :--------------------------------- |
| 1    | 查询Redis缓存 | 优先从缓存获取短链接信息           |
| 2    | 缓存命中      | 直接使用缓存数据，不查数据库       |
| 3    | 缓存未命中    | 查询MySQL数据库                    |
| 4    | 写入缓存      | 将查询结果写入Redis，设置TTL=1小时 |
| 5    | 缓存更新      | 短链接信息变更时主动删除缓存       |

**日志记录流程（异步）：**

| 步骤 | 动作           | 说明                     |
| :--- | :------------- | :----------------------- |
| 1    | 获取访客IP     | 从请求头获取真实IP       |
| 2    | 获取User-Agent | 解析浏览器和操作系统信息 |
| 3    | 获取Referer    | 获取来源页面地址         |
| 4    | 识别设备类型   | PC / 手机 / 平板         |
| 5    | 组装日志数据   | 生成日志记录             |
| 6    | 发送到消息队列 | 异步处理，不阻塞跳转     |
| 7    | 批量写入数据库 | 定时批量写入，减少IO     |

#### 5.4.7 输出结果

| 结果类型 | 输出                   |
| :------- | :--------------------- |
| 成功跳转 | 302重定向到原始长链接  |
| 密码验证 | 返回密码输入页面       |
| 错误情况 | 返回对应的错误提示页面 |

#### 5.4.8 页面与交互要求

**密码输入页面：**

- 简洁的密码输入表单
- 居中显示，移动端适配
- 输入框自动聚焦
- 支持回车提交
- 密码错误时显示错误提示

**错误页面：**

- 404页面：链接不存在
- 过期页面：链接已过期
- 禁用页面：链接已被禁用

**跳转性能：**

- 跳转延迟 < 100ms
- 不展示中间页面，直接跳转

#### 5.4.9 业务规则

| 规则         | 说明                                      |
| :----------- | :---------------------------------------- |
| 重定向类型   | 使用302临时重定向（支持后续变更目标地址） |
| 缓存策略     | 热点链接缓存在Redis，TTL=1小时            |
| 日志异步     | 访问日志异步写入，不影响跳转性能          |
| 防刷机制     | 同一IP每秒超过10次请求时，临时限流        |
| 密码错误限制 | 同一短码连续5次密码错误，锁定5分钟        |

#### 5.4.10 异常场景

| 场景         | 处理方式              | 展示内容                          |
| :----------- | :-------------------- | :-------------------------------- |
| 短码不存在   | 返回404页面           | “链接不存在或已被删除”            |
| 短链接已禁用 | 返回禁用页面          | “该链接已被禁用”                  |
| 短链接已过期 | 返回过期页面          | “该链接已超过有效期”              |

### 5.6 数据统计

### 5.7 个人中心

#### 5.7.1. 功能说明

用户查看和管理自己的账号信息，包括个人资料、账号安全设置、套餐信息等。用户可以在个人中心修改密码、查看当前套餐、升级会员等。

**解决的问题：**

- 用户自主管理账号信息
- 查看当前套餐和使用情况
- 修改密码保障账号安全
- 升级会员获取更多功能

#### 5.7.2 使用角色

| 角色       | 说明           |
| :--------- | :------------- |
| 已登录用户 | 所有已登录用户 |

#### 5.7.3 触发方式

| 触发方式                 | 说明                   |
| :----------------------- | :--------------------- |
| 点击右上角头像/昵称      | 下拉菜单选择“个人中心” |
| 点击侧边栏“个人中心”菜单 | 导航菜单入口           |

#### 5.7.4 前置条件

| 条件       | 说明               |
| :--------- | :----------------- |
| 用户已登录 | 未登录用户无法访问 |

#### 5.7.5 输入项

**个人资料模块：**

| 字段     | 是否必填 | 说明         | 规则                                       |
| :------- | :------- | :----------- | :----------------------------------------- |
| 头像     | 否       | 用户头像     | 支持图片上传；建议尺寸200x200；格式jpg/png |
| 昵称     | 否       | 显示名称     | 长度2-20字符；不可包含特殊字符             |
| 邮箱     | 是       | 登录账号     | 不可修改，只读显示                         |
| 注册时间 | 是       | 账号创建时间 | 只读显示                                   |

**账号安全模块：**

| 字段       | 是否必填 | 说明       | 规则                           |
| :--------- | :------- | :--------- | :----------------------------- |
| 当前密码   | 是       | 验证身份   | 修改密码时必填                 |
| 新密码     | 是       | 新登录密码 | 长度6-20位；至少包含字母和数字 |
| 确认新密码 | 是       | 重复新密码 | 必须与新密码一致               |

**套餐信息模块：**

| 有效期   | 套餐到期时间           |
| 本月已用 | 本月已生成的短链接数量 |
| 本月限额 | 本月可生成的总数量     |
| 使用率   | 已用/限额，进度条展示  |

| 步骤 | 动作               | 说明                 |
